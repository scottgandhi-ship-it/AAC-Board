## Executive Summary

A standalone marketing website that positions Scott's AAC Board as the affordable, cross-platform alternative to $250+ AAC apps. The site must convert three audiences: parents discovering AAC for the first time, therapists/SLPs evaluating tools, and supporters who want to donate or request features. Native app store links for iOS and Android are primary CTAs.

## 1. Competitive Website Analysis -- What AssistiveWare Does Well

### Their Website Structure

| Page/Section | Purpose | What Works |
|-------------|---------|------------|
| Homepage | Mission-first ("What will you say?") | Leads with emotion, not product features |
| /products/proloquo2go | Product deep dive | Clear feature breakdown with screenshots |
| /products/proloquo | Next-gen product | Comparison tools to help users choose |
| /learn-aac | Educational hub | 15+ guides on AAC best practices |
| /blog | Content marketing | User stories, self-advocate voices, tips |
| Core Word Classroom | Free resources | Downloadable planners, printable boards |
| /support | Help center | Product-specific help hubs |
| /about | Company story | Mission-driven, founder story, team bios |

### What We Can Learn From Them

1. **Lead with mission, not features** -- "Everyone has something to say" lands harder than "20,000 symbols"
2. **Content is their moat** -- Learn AAC guides, Core Word Classroom, blog = SEO + trust
3. **Segment by audience** -- They speak differently to parents vs SLPs vs schools
4. **Social proof through user voices** -- Nonspeaking bloggers and self-advocates create authenticity
5. **Free resources drive traffic** -- Printable boards and planners bring people to the site organically

### Where We Can Outperform Them

1. **Price transparency** -- They bury pricing. We lead with "$9.99" as a differentiator.
2. **Platform inclusivity** -- They're Apple-only. We're everywhere.
3. **Community-driven development** -- No competitor has public feature voting or donation transparency.
4. **Modern design** -- Their site feels clinical. Ours can feel warm, modern, and parent-friendly.
5. **Speed to value** -- Show the app working in seconds, not paragraphs.

### 2F. Community Section

#### Feature Request Board

**Goal**: Community-driven development. No competitor has this.

**Features**:
- Public board showing requested features
- Anyone can submit a request (with email)
- Upvote/downvote system
- Status labels: Requested, Under Review, Planned, In Progress, Shipped
- Admin can respond to requests
- Most-voted features surface to top

**Implementation options**:
- Canny (canny.io) -- purpose-built feature request tool. Free tier: 1 board, 100 tracked users. Pro: $79/mo.
- Nolt (nolt.io) -- simpler alternative. $29/mo.
- GitHub Discussions -- free, but requires GitHub account (friction for parents)
- Custom build -- more work but no ongoing cost

**Recommendation**: Start with Canny free tier. Migrate to custom or upgrade if we outgrow it.

#### Donation / Support Us Page

**Goal**: Let supporters fund development beyond the $9.99 purchase

**Important context for app stores**:
- Apple and Google take 30% of app store purchases
- Donations on the WEBSITE bypass the app store cut entirely
- You CANNOT solicit donations inside the iOS/Android app (Apple ToS violation)
- The website is the right place for this

**Donation options**:

| Platform | Fee | Recurring? | Pros | Cons |
|----------|-----|-----------|------|------|
| Stripe | 2.9% + $0.30 | Yes | Most professional, full control, custom amounts | Requires backend or Stripe Payment Links |
| Ko-fi | 0% (Gold: $6/mo for extras) | Yes | Zero fee, simple embed, no backend needed | Less professional, limited customization |
| Buy Me a Coffee | 5% | Yes | Simple, well-known | Higher fee, less control |
| GitHub Sponsors | 0% | Yes | Zero fee, dev-friendly | Requires GitHub account (friction for parents) |
| Open Collective | 5-10% | Yes | Full transparency, fiscal hosting | Higher fees |

**Recommendation**: **Stripe Payment Links + Ko-fi embed**

- **Stripe Payment Links** for the primary donation system: Create preset amounts ($5, $10, $25, $50, custom) as hosted payment links. No backend required. Professional, trustworthy, supports recurring. Revenue goes directly to you minus 2.9% + $0.30.
- **Ko-fi widget** as a secondary "quick donate" option embedded on multiple pages. Zero platform fee. Lower friction for small donations.

**Donation tiers** (suggested):
- $5 -- "Buy us a coffee" (supporter)
- $10 -- "Sponsor a feature vote" (your vote counts double)
- $25 -- "Give a child a voice" (sponsor a free license for a family in need)
- $50+ -- "Champion" (name on Wall of Supporters + early access to new features)

**Transparency section**:
- Show how donations are used: "X% development, X% hosting, X% accessibility testing"
- Running total of donations received (updated monthly)
- Link donations to shipped features: "This feature was funded by community donations"

## 3. Tech Stack Recommendation

### Recommended: Static Site (HTML/CSS/JS) + Firebase Hosting

**Why**:
- You already have Firebase experience (NoteForged)
- Free SSL, custom domain, CDN, fast deploys
- No framework overhead -- matches your single-file philosophy
- Static pages are fast (Lighthouse 100 performance)
- Firebase can handle contact forms via Cloud Functions if needed
- Easy to add a blog later (markdown -> HTML build step)

**Alternatives considered**:

| Option | Pros | Cons |
|--------|------|------|
| GitHub Pages | Free, simple | No server-side anything, slower deploys |
| Squarespace/Wix | Fast to launch, templates | Monthly cost ($16-33/mo), less control, harder to customize |
| Next.js/Astro | Modern frameworks, great DX | Overkill for a marketing site, adds build complexity |
| WordPress | Blog-native, tons of plugins | Hosting cost, security maintenance, PHP |

**Decision**: Firebase Hosting with static HTML/CSS/JS. Add Firebase Cloud Functions only if needed for contact forms or donation webhooks.

- Emotional state: Professional, evaluating options for clients

- How they find us: Professional networks, ASHA resources, Google

**Tertiary: Supporters and advocates**
- Emotional state: Passionate about accessibility and inclusion
- Need: A cause to support, transparency about impact
- How they find us: Social media, word of mouth, disability advocacy communities
- What converts them: Mission, transparency, donation impact visibility

## 5. Payment and Donation Architecture

### App Store Purchases (the app itself)

- **iOS**: Apple App Store at $9.99 (Apple takes 30% = $3.00, you get $6.99)
- **Android**: Google Play Store at $9.99 (Google takes 30% initially, 15% after $1M revenue)
- Note: Apple reduced small business commission to 15% if annual revenue is under $1M

**Net revenue per sale**:
- iOS (small business): ~$8.49 per sale
- Android (small business): ~$8.49 per sale

### Website Donations (bypasses app store cut)

- **Stripe Payment Links**: Preset amounts, custom amounts, recurring options
- **Ko-fi embed**: Quick small donations
- Donations are NOT subject to app store commission (30%)
- IMPORTANT: Do NOT mention donations or link to website donation page from inside the iOS app (Apple ToS violation)

### Future Premium Features (if needed)

- If we add premium cloud TTS voices as a subscription, this happens via in-app purchase (app store handles billing)
- Apple/Google take 30% (15% small business) of subscription revenue
- Alternative: Offer premium features via website purchase + license key (avoids app store cut but adds complexity)
