## What's Needed: SLP Session Report

### The Problem

SLPs need to document AAC usage for IEPs, progress reports, and session notes. The current CSV export is raw data (one row per tap) that requires manual analysis. SLPs need a **formatted summary** they can share, print, or paste into clinical documentation.

### Approach

Add a "Share Progress Report" button in the Insights section that generates a formatted text report and shares it via navigator.share() (or copies to clipboard as fallback). No PDF generation needed -- plain text is the most portable and paste-friendly format for clinical documentation systems.

### Report Content

Header:
- Child's name (if set) or "AAC Board Progress Report"
- Date range (configurable: last 7 days, 30 days, 90 days, or custom)
- Generated date

Communication Summary:
- Total words communicated
- Average words per day
- Day streak (consecutive days with usage)
- Total sessions (if activity data exists)

Top Vocabulary (top 15 words):
- Word, category (color/type), tap count
- Formatted as a simple table

Category Breakdown:
- Taps by Fitzgerald Key category (verbs, nouns, pronouns, etc.)
- Shows communication balance
- Steve review: Look up button color from templates/buttons array. Buttons no longer in current template fall under "Other"

Activity Usage (if any):
- Activities used, session count, avg duration
- Top words per activity
- Word coverage percentage

Weekly Trend:
- Last 4-8 weeks of daily averages
- Simple text-based trend (increasing, stable, decreasing)

### Files to Modify

- `index.html` -- new button in Insights section, report generation function, date range picker

### Implementation

**Step 1: Add UI to Insights section**

After the existing "Export CSV" button, add:
- "Share Progress Report" button (styled like export buttons)
- Date range selector: "Last 7 days" / "Last 30 days" / "Last 90 days"

**Step 2: Report generation function**

`generateSLPReport(days)`:
- Pull from existing usage data (usageLog, usageSummary, activitySessions)
- Filter to requested date range using `Date.now() - (days * 86400000)` inclusive boundary (Steve review: matches existing trimUsageLog pattern)
- Compute all metrics using existing helper functions (getWordFrequencies, getDailyWordCounts, getActivityInsights)
- Format as plain text with clear sections
- Include child's name from localStorage if set

**Step 3: Share/copy**

- Use navigator.share() with the report as text (same pattern as CSV export)
- Fallback: copy to clipboard with toast confirmation
- Filename: `{childName}-progress-{dateRange}-{date}.txt`

### Edge Cases

- No usage data: Show message "No usage data recorded yet" instead of empty report
- No child name: Use "AAC Board Progress Report" as header
- No activity data: Omit activity section entirely
- Very few words (<10): Still generate report, just shorter

### Acceptance Criteria

- "Share Progress Report" button visible in Insights (parent mode only)
- Date range selector works (7/30/90 days)
- Report generates with all sections populated from real data
- Empty sections omitted gracefully
- Share via navigator.share() on iOS, clipboard fallback on web
- Child's name included when set
- Category breakdown maps to Fitzgerald Key colors/types
- Report is paste-friendly plain text (no HTML, no special chars)
