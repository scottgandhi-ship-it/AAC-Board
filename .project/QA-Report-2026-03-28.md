### Fitzgerald Key Colors (tokens.css)
All category colors are light pastels designed as button backgrounds with dark text (#364153). The background-to-white ratios fail 4.5:1 but this is by design -- the actual text contrast (dark on light) likely passes. **Needs manual verification on device.**

## ISSUES REQUIRING ACTION

### HIGH Priority
1. **File sync**: `js/storage.js` out of sync with `www/js/storage.js` -- run `.scripts/sync-all.sh`
2. **Guiding Steps: No privacy statement** -- needs visible privacy disclosure for parents (required for App Store Kids category)
3. **Focus indicator removed**: `.walkthrough-step-input:focus` in `activities.css` sets `outline: none` -- needs visible focus style

### MEDIUM Priority
4. **Spanish tilde missing**: ~11 words in `js/i18n.js` use 'n' instead of 'n-tilde' (bano, panal, muneca, pequeno)
5. **Guiding Steps: No Yappie cross-promotion** -- companion app not promoted despite being noted in launch plan
6. **Modal focus trapping**: Modals have `aria-modal="true"` but no explicit JS focus trap
7. **Keyboard grid navigation**: No arrow key support for navigating vocabulary grid cells

### LOW Priority
8. **Form labels**: Some inputs use `aria-label` instead of semantic `<label for="">` elements
9. **Fitzgerald Key color contrast**: Verify actual text-on-background contrast on device (likely OK but needs confirmation)
10. **Guiding Steps accessibility**: Full WCAG audit needed (single 413KB file not fully audited)
